🔐 AUTH - Autenticación y Autorización

Guía de Desarrollo de User Stories

Español (original) Português (pt-BR)

📋 Vista General

Propósito: El módulo AUTH es el núcleo de seguridad del sistema CardDemo. Gestiona la autenticación de usuarios, autorización basada en roles, y seguridad de sesiones.

Contexto de Negocio: Provee control de acceso seguro para usuarios administrativos y de back-office, asegurando que cada usuario tenga acceso únicamente a las funcionalidades permitidas según su rol.

Ubicación en el código:

  • /app/features/auth/authSlice.ts - Redux state management
  • /app/services/authApi.ts - API services
  • /app/components/auth/ProtectedRoute.tsx - Protección de rutas
  • /app/hooks/useSecureSession.ts - Gestión de sesión
  • /app/pages/LoginPage.tsx - Página de login

🎯 Guía de Desarrollo de User Stories

📝 Plantillas Específicas de US

Login de Usuario

Como usuario del sistema, quiero poder iniciar sesión con mis credenciales para acceder a las funcionalidades según mi rol

Criterios de Aceptación:
  • El formulario muestra campos para User ID y Password
  • Los campos aceptan máximo 8 caracteres
  • La entrada se convierte automáticamente a mayúsculas
  • Se valida que ambos campos estén completos
  • Se muestra mensaje de error si credenciales inválidas
  • Después de login exitoso, se redirige según el rol
  • Se almacenan tokens de manera segura en localStorage
Simple (1-2 pts)
Protección de Rutas por Rol

Como administrador del sistema, quiero que las rutas estén protegidas según roles para asegurar que los usuarios solo accedan a funciones permitidas

Criterios de Aceptación:
  • Las rutas requieren autenticación para acceder
  • Se verifica el rol del usuario para rutas con restricción
  • Usuarios no autenticados redirigidos a login
  • Usuarios con rol incorrecto redirigidos a su menú
  • Se mantiene URL destino para redirección después de login
Medio (3 pts)
Gestión de Sesión Segura

Como usuario autenticado, quiero que mi sesión expire automáticamente después de inactividad para proteger mi cuenta contra acceso no autorizado

Criterios de Aceptación:
  • La sesión tiene duración máxima de 8 horas
  • Se verifica la expiración cada 5 minutos
  • El token se renueva automáticamente si está próximo a expirar
  • Al expirar la sesión, se hace logout automático
  • Se limpia información al cerrar ventana/pestaña
  • Se verifica sesión al cambiar visibilidad de pestaña
Medio (3-5 pts)
Renovación Automática de Token

Como sistema, quiero renovar automáticamente los tokens de acceso para mantener la sesión del usuario activa sin interrupciones

Criterios de Aceptación:
  • El token se renueva cada 5 minutos mientras sesión activa
  • Se usa refresh token para obtener nuevo access token
  • Si falla renovación, se hace logout automático
  • Renovación no interfiere con experiencia del usuario
  • Nuevo token reemplaza al anterior en localStorage
Medio (3-5 pts)

⚡ Factores de Aceleración de Desarrollo

🔧 ProtectedRoute Component

Reutilización: Componente listo para proteger cualquier ruta

Uso típico:

<ProtectedRoute requiredRole="admin"> <AdminPage /> </ProtectedRoute>

Aceleración: Protección de rutas sin escribir lógica adicional

🎣 useSecureSession Hook

Reutilización: Hook con toda la lógica de gestión de sesión

Métodos disponibles:

  • checkSessionExpiry() - Verificar si sesión está vigente
  • attemptTokenRefresh() - Renovar token manualmente
  • clearSession() - Limpiar sesión y redirigir
  • performImmediateLogout() - Logout inmediato

Aceleración: Gestión completa de sesión con una línea

📦 authSlice (Redux)

Reutilización: Redux slice completo con todas las acciones

Acciones disponibles:

  • loginUser - Autenticar usuario
  • logoutUser - Cerrar sesión
  • refreshToken - Renovar access token
  • validateToken - Validar token JWT
  • immediateLogout - Logout sin confirmación

Aceleración: Estado global de auth completamente gestionado

🌐 authApi Service

Reutilización: Servicio con todos los endpoints de autenticación

Endpoints:

  • POST /auth/login - Login
  • POST /auth/logout - Logout
  • POST /auth/refresh - Refrescar token
  • POST /auth/validate - Validar token
  • GET /auth/health - Health check

Aceleración: API client configurado y tipado

🔧 Fundamentos Técnicos

🏗️ Componentes Clave

authSlice.ts

Ubicación: /app/features/auth/authSlice.ts

Responsabilidad: Gestión del estado de autenticación en Redux

Estado gestionado:

  • user - Usuario autenticado actual
  • token - Access token JWT
  • isAuthenticated - Estado de autenticación
  • isLoading - Indicador de carga
  • error - Errores de autenticación

authApi.ts

Ubicación: /app/services/authApi.ts

Responsabilidad: Comunicación con endpoints de autenticación

Tipos definidos:

  • AuthResponse - Respuesta de login
  • TokenValidationResponse - Respuesta de validación

ProtectedRoute.tsx

Ubicación: /app/components/auth/ProtectedRoute.tsx

Responsabilidad: Proteger rutas según autenticación y roles

Props:

  • children - Componente a proteger
  • requiredRole (opcional) - Rol requerido: 'admin' | 'back-office'

useSecureSession.ts

Ubicación: /app/hooks/useSecureSession.ts

Responsabilidad: Gestión segura de sesión con validaciones automáticas

Características:

  • Verificación de expiración (8 horas máximo)
  • Renovación automática cada 5 minutos
  • Limpieza al cerrar ventana
  • Validación al cambiar visibilidad de pestaña

LoginPage.tsx

Ubicación: /app/pages/LoginPage.tsx

Responsabilidad: Interfaz de usuario para autenticación

Características:

  • Formulario con validación de campos
  • Conversión automática a mayúsculas
  • Límite de 8 caracteres por campo
  • Visualización de contraseña
  • Manejo de errores amigable
  • Redirección según rol

🔗 Dependencias

Dependencias Internas:
  • Store (Redux): Para gestión de estado global

📋 Reglas de Negocio

Autenticación

  • User ID: Máximo 8 caracteres, obligatorio, convertido a mayúsculas
  • Password: Máximo 8 caracteres, obligatorio, convertido a mayúsculas
  • Roles: Admin (userType: 'A') o Back-office (userType: 'U')
  • Tokens: Access token válido por 1 hora, refresh token por duración de sesión

Sesión

  • Duración máxima: 8 horas de sesión continua
  • Verificación: Cada 5 minutos
  • Renovación: Automática si token próximo a expirar
  • Expiración: Logout automático después de 8 horas o al cerrar ventana

Autorización

  • Rutas protegidas: Requieren autenticación
  • Rutas con rol: Verifican rol específico
  • Redirección: Admin → /menu/admin, Back-office → /menu/main
  • Sin acceso: Redirige a login o menú correspondiente

🔒 Consideraciones de Seguridad

✅ Mejores Prácticas Implementadas

  • Almacenamiento Seguro: Tokens en localStorage con limpieza completa al logout
  • Validación Continua: Verificación de token en cada ruta protegida
  • Expiración Automática: Sesiones expiran después de 8 horas
  • Renovación Automática: Tokens se renuevan cada 5 minutos
  • Logout Inmediato: Si token inválido o sesión expirada
  • Limpieza al Cerrar: SessionStorage limpiado al cerrar ventana

⚠️ Mejoras Futuras (Tech Debt)

  • HttpOnly Cookies: Migrar tokens a cookies HttpOnly para mayor seguridad
  • CSRF Protection: Implementar tokens CSRF
  • Rate Limiting: Limitar intentos de login (prevenir fuerza bruta)
  • 2FA: Autenticación de dos factores
  • Biometric Auth: Autenticación biométrica para dispositivos móviles
  • Password Policies: Políticas de contraseña más estrictas
